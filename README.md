# 🐀💥 Крысиная возня

Карточная игра — Telegram Mini App. PvE против ИИ-ботов (механика «Exploding Kittens»), онлайн-лидерборд топ-500 на Supabase, деплой на GitHub Pages.

Колода — авторский набор **«Крысиная возня»**: 110 карт, у каждой уникальный арт (сгенерирован из PDF-исходников в `public/assets/cards/`, WebP). Состав: 8 «Смертельный кринж», 10 «Крыса, живи!», по 10 «Гоп-стоп» / «Чилл» / «Разрешите доебаца» / «Крысиная суета» / «Бухгалтерия интересуется» / «Пидора ответ» и 8 видов крысокарт по 4. Для партий 2–5 игроков добор автоматически урезается до классической плотности (случайное подмножество — арты ротируются между партиями).

**Игра полностью играбельна сразу после клонирования** — без Supabase (лидерборд уходит в локальный fallback); при отсутствии файла арта рендерится CSS-плейсхолдер.

## Стек

- **Фронтенд:** React + Vite + TypeScript, Zustand (стейт), Framer Motion (анимации), CSS Modules + дизайн-токены ([src/styles/tokens.css](src/styles/tokens.css)). Токены — CSS-переменные, поэтому тема синхронизируется с `Telegram.WebApp.themeParams` в рантайме.
- **Бэкенд:** Supabase — Postgres (таблица `leaderboard`, RLS: читать всем, писать никому) + Edge Function `verify-and-submit` (валидация Telegram initData + серверный расчёт очков).
- **Звук:** Web Audio API, все эффекты синтезируются кодом; свои файлы подхватываются автоматически.

## 1. Локальный запуск

```bash
npm install
npm run dev      # http://localhost:5173/exploding-rats/
npm test         # юнит-тесты движка и ИИ (38 тестов + симуляции партий)
```

Вне Telegram игра работает с мок-пользователем «Гость» — удобно для разработки.

## 2. Supabase (онлайн-лидерборд)

1. Создайте проект на [supabase.com](https://supabase.com).
2. SQL Editor → выполните целиком [supabase/sql/schema.sql](supabase/sql/schema.sql) (таблица + индекс + RLS).
3. Установите [Supabase CLI](https://supabase.com/docs/guides/cli) и задеплойте функцию:
   ```bash
   supabase login
   supabase link --project-ref <PROJECT_REF>
   supabase functions deploy verify-and-submit
   ```
4. Задайте секрет с токеном бота (токен даёт @BotFather; **в репозиторий его класть нельзя**):
   ```bash
   supabase secrets set BOT_TOKEN=123456:ABC-DEF...
   ```
5. Впишите публичные ключи в [src/config.ts](src/config.ts):
   ```ts
   export const SUPABASE_URL = 'https://<PROJECT_REF>.supabase.co';
   export const SUPABASE_ANON_KEY = '<anon public key>'; // Settings → API
   ```

Без этих шагов игра работает офлайн: лидерборд считается локально (та же формула очков) и показывает плашку «Онлайн-таблица недоступна».

**Безопасность:** запись в таблицу возможна только через Edge Function (service role). Функция проверяет подпись `initData` (HMAC-SHA256, секрет = `HMAC("WebAppData", BOT_TOKEN)`), свежесть `auth_date` (≤10 мин), rate-limit (1 сабмит в 20 сек) и пересчитывает очки сама — клиентским числам не доверяет.

## 3. Деплой на GitHub Pages

1. Создайте репозиторий и запушьте код в ветку `main`.
2. GitHub → Settings → Pages → Source: **GitHub Actions**.
3. Workflow [.github/workflows/deploy.yml](.github/workflows/deploy.yml) при пуше в `main` прогоняет тесты, собирает и деплоит. `base` подставляется автоматически из имени репозитория; для локальной сборки поменяйте default в [vite.config.ts](vite.config.ts), если репозиторий называется не `exploding-rats`.
4. URL приложения: `https://<username>.github.io/<repo>/`.

## 4. Регистрация Mini App у @BotFather

1. `/newbot` → получите бота и токен (этот токен — в секреты Supabase, шаг 2.4).
2. `/newapp` → выберите бота → укажите название, описание, картинку и **Web App URL** = URL с GitHub Pages.
3. Готово: игра открывается по ссылке `t.me/<bot>/<appname>` и через кнопку меню бота.

## 5. Картинки карт

Арты уже лежат в `public/assets/cards/` (WebP 630×880, сконвертированы из PDF-исходников набора). **У каждого экземпляра карты свой арт**: файл называется `<тип>-<номер>.webp`, номер = 1..количество карт типа.

| Основа имени | Карта | Кол-во |
|---|---|---|
| `exploding-rat-1…8` | Смертельный кринж | 8 |
| `defuse-1…10` | Крыса, живи! | 10 |
| `attack-1…10` | Гоп-стоп | 10 |
| `skip-1…10` | Чилл | 10 |
| `favor-1…10` | Разрешите доебаца | 10 |
| `shuffle-1…10` | Крысиная суета | 10 |
| `see-the-future-1…10` | Бухгалтерия интересуется | 10 |
| `nope-1…10` | Пидора ответ | 10 |
| `rat-1-1…4` … `rat-8-1…4` | 8 видов крысокарт | 8×4 |
| `card-back` | Рубашка | 1 |

Чтобы заменить арт, положите файл с тем же именем (`.webp`, либо `.png` — он подхватится как fallback, если webp нет). Рекомендуемое разрешение 630×880+ (пропорции покерной карты). Манифест карт (названия, количество, описания, основы имён) — в [src/data/cards.ts](src/data/cards.ts).

## 6. Как добавить свои звуки

Положите файлы в `public/assets/sfx/` с именами `play`, `draw`, `explosion`, `defuse`, `win`, `lose`, `tap`, `steal`, `shuffle`, `nope` и расширением `.mp3`/`.wav`/`.ogg`. Пока файла нет — звук синтезируется Web Audio API.

## Мультиплеер

Онлайн-игра 2–5 человек (включая дуэль 1 на 1) на **Supabase Realtime** — отдельного сервера не нужно, достаточно уже настроенного проекта Supabase.

- Меню → «Мультиплеер» → «Создать комнату» → код из 4 букв, им делятся с друзьями (тап по коду — шэринг в Telegram).
- Модель — **хост-авторитарная**: движок исполняется у создателя комнаты, гости шлют действия (`play/draw/nope/...`), хост валидирует их через движок и рассылает снапшоты с событиями (звук/анимации у всех).
- Окно «Пидора ответа» — 3 секунды, каждая новая карта в цепочке продлевает его.
- Таймауты: 30 с на «отдай карту»/«верни кринж», 60 с на ход — партия не зависает из-за уснувшего игрока (авто-действие).
- Дисконнект гостя ≥7 с → он выбывает (карты в сброс); реконнект в течение grace-периода безопасен, «догоняющий» снапшот приходит по `hello`. Выход хоста завершает партию.
- Очки рейтинга в мультиплеере не начисляются (сервер не может верифицировать исход между клиентами) — режим казуальный. Начисление потребует переноса движка в Edge Function (следующий шаг, код движка уже чистый TypeScript без DOM).

## Правила и очки

Полные правила — на экране «Правила» внутри игры. Очки за победу (поражение = 0, но `games_played +1`):

```
pointsThisGame = round((100 + боты×20) × множитель_сложности + min(серия, 10)×15)
множитель: easy 1.0 · medium 1.5 · hard 2.5
```

  store/                Zustand: настройки + оркестратор партии
supabase/
  sql/schema.sql        таблица + RLS
  functions/verify-and-submit/   валидация initData + серверный счёт
